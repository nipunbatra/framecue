What it does

Everything renders inside the browser tab — nothing extra to install.

Auto-matched files

Pick a folder; each video finds its same-named .vtt subtitles and .json/.txt metadata automatically.

Constant metadata

A persistent overlay (title, date, anything) stays on screen for the whole clip — perfect for context during a lecture or talk.

Synced subtitles

Custom WebVTT renderer styled for readability on a shared screen. Resize and reposition on the fly.

Live timecode

An always-visible current / duration readout for the audience.

Presentation mode

One key hides all the chrome so the video fills the tab — clean to share.

100% local & private

No uploads, no server, no dependencies. Your files never leave your machine.

How it works

Name the files with the same base name and drop them in a folder:

my-folder/
├── talk.mp4      ← the video
├── talk.vtt      ← subtitles  (optional)
└── talk.json     ← metadata overlay  (optional, or talk.txt)

Metadata as JSON renders as key: value lines:

{
  "Lecture": "Introduction to Machine Learning, 14",
  "Lecture Date": "20 Feb 2024"
}

A plain talk.txt is shown as-is. Then:

  1. Open the app and click Pick folder (or drag a folder onto the page).
  2. Click a video in the sidebar — subtitles and metadata load automatically.
  3. Share the Chrome Tab in Meet/Zoom and tick “Also share tab audio.”
Playback uses the browser's codecs — MP4 (H.264) and WebM are the safe bets. .mkv/.avi may not decode in Chrome.

Run it locally

No build step. Two ways:

  1. Use it online — just launch the hosted app (file picking works over HTTPS too).
  2. Or clone and open the file directly:
git clone https://github.com/nipunbatra/video-subtitle-overlay
open video-subtitle-overlay/app.html      # macOS
